# Flutter Skill Router

Use this for Flutter-specific work: widget architecture, navigation, state management, theming, a11y, performance, and iOS/Android platform behavior.

## When to load

- You are building or refactoring a Flutter app.
- You need platform UX guidance (Material vs Cupertino).
- You need patterns for state, navigation, and design systems.
- You need widget/integration test strategy.

## When NOT to load

- You are only writing Dart language code (use `../dart/SKILL.md`).
- You are doing backend API design (use `../api/SKILL.md`).
