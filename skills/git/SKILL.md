# Git Workflow

## Branching

- Use prefixes: `feature/`, `fix/`, `refactor/`, `docs/`
- Keep branches short-lived

## Commits

- One logical change per commit
- Message: what + why
- Avoid `--amend` unless explicitly requested

## PRs

- Include summary, test plan, risks
- Link issues/tickets when applicable
