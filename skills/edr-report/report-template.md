# End of Day Report Template

## When to load

- You need to produce a same-day shift report.
- You have partial notes and need a complete manager-ready write-up.
- You must follow a fixed template from the requester.

## When NOT to load

- The request is for a non-daily format (weekly recap, retrospective, release notes).

## Identity defaults

Always use these values — never leave them as placeholders:

- **NAME**: Danilo Casim Jr
- **POSITION**: IT Intern

## Core rules

- Always fill NAME as `Danilo Casim Jr` and POSITION as `IT Intern`.
- Keep the requester's template structure and labels exactly.
- Use factual, verifiable statements based on provided notes or repository evidence.
- Use same-day commit messages as the primary evidence for `END OF DAY REPORT` accomplishments.
- If data is unknown, keep placeholders in brackets.
- Keep tone professional, concise, action-oriented, and non-technical.
- Write `NEXT STEPS` as concrete actions for the next shift.
- Do not include commit IDs in the report.
- Keep section headers clean; do not append extra helper text under headers.

## Canonical template

```text
NAME: Danilo Casim Jr
POSITION: IT Intern
DATE: <YYYY-MM-DD>
TIME IN: <HH:MM or [TIME IN]>
BREAK: <HH:MM-HH:MM or [BREAK]>
TIME OUT: <HH:MM or [TIME OUT]>
TOTAL OF HOURS: <computed or [TOTAL OF HOURS]>

END OF DAY REPORT
- <accomplishment 1>
- <accomplishment 2>
- <accomplishment 3>

NEXT STEPS:
- <next action 1>
- <next action 2>
- <next action 3>

NEXT SHIFT:
- <date/time or [NEXT SHIFT DATE/TIME]>
```

## Minimal examples

```text
END OF DAY REPORT
- Verified branch status and repository cleanliness.
- Reviewed outstanding tasks with mentor and documented blockers.
- Prepared next-shift implementation checklist.

NEXT STEPS:
- Continue assigned feature implementation.
- Open/prepare PR with clear commit messages.
- Validate output and update handoff notes.
```

## Anti-patterns

- Inventing commits, PRs, or deliverables not present in evidence.
- Rewriting the template headings when the requester asks for exact format.
- Long narrative paragraphs instead of scannable bullet points.

## Checklist

- Template fields are present and in original order.
- Unknown values remain placeholders.
- Accomplishments are evidence-based.
- Next steps are specific and measurable.
